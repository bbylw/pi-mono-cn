设计理念

Pi 具有极强的可扩展性，因此不必规定你的工作流程。

没有 MCP

使用 README 构建 CLI 工具，或构建添加 MCP 支持的扩展。

没有子智能体

通过 tmux 生成 pi 实例，或使用扩展构建自己的子智能体。

没有权限弹出窗口

在容器中运行，或使用扩展构建你自己的确认流程。

没有计划模式

将计划写入文件，或使用扩展构建，或安装包。

没有内置待办事项

它们会混淆模型。使用 TODO.md 文件，或使用扩展构建。

没有后台 bash

使用 tmux。完全可观察性，直接交互。

Pi 包生态

通过 npm 或 git 打包和分享扩展、技能、提示和主题。社区驱动的生态系统。

安全提示

Pi 包以完全系统访问权限运行。在安装第三方包之前，请查看源代码。

$ pi install npm:@foo/pi-tools $ pi install git:github.com/user/repo $ pi list
package.json

创建你的包

通过在 package.json 中添加 pi 键来创建包：

{ "name": "my-pi-package", "keywords": ["pi-package"], "pi": { "extensions": ["./extensions"], "skills": ["./skills"], "prompts": ["./prompts"], "themes": ["./themes"] } }
